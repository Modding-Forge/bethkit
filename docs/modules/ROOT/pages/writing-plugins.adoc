= Writing & Patching Plugins

bethkit provides two complementary APIs for producing plugin files:

* `PluginWriter` — build a new plugin from scratch
* `PluginPatcher` — replace specific records inside an existing plugin while copying
  everything else verbatim

== PluginWriter — build from scratch

`PluginWriter` constructs a valid plugin file byte-by-byte.
It handles the TES4 header, GRUP wrappers, record headers, and group-size fields automatically.

=== Minimal example

[source,rust]
----
use bethkit_core::{
    FormId, GameContext, RecordFlags, Signature,
    WritableGroup, WritableGroupChild,
    WritableRecord, WritableSubRecord, PluginWriter,
};
use std::path::Path;

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let mut writer = PluginWriter::new(GameContext::sse(), 1.71);

    // Add master files
    writer.add_master("Skyrim.esm");
    writer.set_description("A demonstration plugin.");

    // Build a KYWD group with one record using struct literals.
    let subrecord = WritableSubRecord {
        signature: Signature(*b"EDID"),
        data: b"MyKeyword\0".to_vec(),
    };
    let record = WritableRecord {
        signature: Signature(*b"KYWD"),
        flags: RecordFlags::empty(),
        form_id: FormId(0x00_000D62),
        form_version: 44,
        subrecords: vec![subrecord],
    };
    let group = WritableGroup {
        label: *b"KYWD",
        group_type: 0,
        children: vec![WritableGroupChild::Record(record)],
    };
    writer.add_group(group);

    writer.write_to_file(Path::new("MyMod.esp"))?;
    Ok(())
}
----

=== Setting the LOCALIZED flag

[source,rust]
----
writer.set_localized(true);
----

When `set_localized` is true the writer sets bit 7 of the TES4 record flags.
String-table generation is your responsibility — see xref:string-tables.adoc[Localized Strings].

=== ESLification

`PluginWriter::eslify` reassigns all plugin-owned FormIDs to the ESL range
(`0xFE_000800`–`0xFE_000FFF`), patches inline FormID references inside subrecord data,
and sets the `LIGHT` flag on the `TES4` header.

[source,rust]
----
use bethkit_core::{GameContext, PluginWriter};
use std::path::Path;

let ctx = GameContext::sse();
let mut writer = PluginWriter::new(ctx, 1.71);
// … populate writer with add_group(…) calls …
writer.eslify()?; // reassigns FormIDs and sets the LIGHT flag

writer.write_to_file(Path::new("MyMod.esl"))?;
----

`eslify()` returns `Err(CoreError::EslRecordLimitExceeded)` if the plugin defines more records
than the game-specific ESL capacity (4 095 for Skyrim SE, 2 047 for other Light-capable games),
or `Err(CoreError::UnsupportedGame)` if the target game does not support
light plugins.

== PluginPatcher — streaming replacement

`PluginPatcher` is designed for automated patchers that change a handful of records in a large
plugin without re-serializing the entire file.

It reads the source plugin sequentially and writes each group/record directly to the output.
Only the records that you have registered for replacement are held in memory simultaneously.
Group sizes are recalculated correctly even when the replacement bytes differ in size from the
original.

=== Basic usage

[source,rust]
----
use bethkit_core::{
    FormId, GameContext, Plugin, PluginPatcher,
    RecordFlags, RecordPatch, Signature, WritableRecord, WritableSubRecord,
};
use std::fs::File;

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let ctx = GameContext::sse();
    let plugin = Plugin::open("MyMod.esp".as_ref(), ctx)?;
    let mut patcher = PluginPatcher::new(&plugin);

    // Build a replacement record — data_size is computed automatically.
    let patch = RecordPatch::from_writable_record(build_replacement());
    patcher.replace_record(FormId(0x00_000ABC), patch);

    let mut out = File::create("MyMod_patched.esp")?;
    patcher.write_to(&mut out)?;
    Ok(())
}

fn build_replacement() -> WritableRecord {
    WritableRecord {
        signature: Signature(*b"KYWD"),
        flags: RecordFlags::empty(),
        form_id: FormId(0x00_000ABC),
        form_version: 44,
        subrecords: vec![
            WritableSubRecord {
                signature: Signature(*b"EDID"),
                data: b"MyReplacementKeyword\0".to_vec(),
            },
        ],
    }
}
----

=== RecordPatch constructors

[cols="1,3",options="header"]
|===
| Constructor | Description

| `RecordPatch::from_writable_record(record)`
| Serialises a `WritableRecord` into a patch.
  `data_size` in the 24-byte header is computed automatically from the
  subrecords.  This is the preferred way to build patches.

| `RecordPatch::from_record_verbatim(record, source)`
| Copies the original bytes of a parsed `Record` from `source` verbatim.
  Returns `None` when no source range is available (i.e. the record was
  constructed in memory, not parsed from a file).

| `RecordPatch::RawBytes(bytes)`
| Low-level escape hatch.  Provide the complete record bytes including the
  24-byte header.  The caller must keep `data_size` and the data block in
  sync.  Use this only when you need full binary control, for example when
  patching a zlib-compressed record.
|===

=== Combining PluginPatcher with PluginWriter

When a patcher needs to insert *new* records rather than replace existing ones, build the new
records with `PluginWriter` first, then use `PluginPatcher` to slot them in alongside the
patched records.

== Performance notes

* `PluginPatcher` cost is O(plugin size) for the copy pass plus O(edits) for the replacements.
  Memory usage is bounded by the size of the replacement bytes you register.
* `PluginWriter` constructs the output in a `Vec<u8>` that grows linearly with the plugin size.
  For very large plugins (100 MB+), write to a `BufWriter<File>` rather than keeping the whole
  output in memory.
